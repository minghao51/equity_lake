# API Keys And Credentials

This project can run in a minimal mode without most third-party credentials.

## Quick Summary

Required only for specific features:

- `FRED_API_KEY`: needed for macro indicator ingestion.
- `FINNHUB_API_KEY`: needed for company news and social sentiment ingestion.
- `REDDIT_CLIENT_ID`, `REDDIT_CLIENT_SECRET`, `REDDIT_USER_AGENT`: needed for the Reddit sentiment loader.
- `SEC_USER_AGENT`: recommended for the SEC filings loader.
- `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`: needed only if your S3 bootstrap bucket is private.

No key required:

- Core US/HK/SG price ingestion through `yfinance`
- China ingestion through existing free-source adapters
- Options flow loader through `yfinance`
- Static dashboard generation

## Environment File

Use the root template:

```bash
cp .env.example .env
```

Then fill only the values you actually need.

## Keys By Feature

### FRED

- Env var: `FRED_API_KEY`
- Used by: macro indicator fetching
- Get it from: [FRED API key page](https://fred.stlouisfed.org/docs/api/api_key.html)

### Finnhub

- Env var: `FINNHUB_API_KEY`
- Used by:
  - `uv run equity-news`
  - `uv run equity-sentiment`
  - ingestion flows that include `us_news` or `us_social_sentiment`
- Get it from: [Finnhub](https://finnhub.io/)

### Reddit

- Env vars:
  - `REDDIT_CLIENT_ID`
  - `REDDIT_CLIENT_SECRET`
  - `REDDIT_USER_AGENT`
- Used by: `reddit_sentiment` loader
- Create credentials at: [Reddit apps](https://www.reddit.com/prefs/apps)

Suggested user agent format:

```text
equity-lake/0.1 (by u/your_reddit_username)
```

### SEC

- Env var: `SEC_USER_AGENT`
- Used by: `sec_filings` loader
- SEC access works best with a descriptive user-agent that includes contact info.

Suggested format:

```text
Equity Lake your-email@example.com
```

### AWS / S3

- Env vars:
  - `AWS_ACCESS_KEY_ID`
  - `AWS_SECRET_ACCESS_KEY`
  - `S3_BUCKET`
- Used by: initial S3 sync if the bucket is private

## CLI Notes

Minimal local workflow with no extra keys:

```bash
uv run equity ingest --markets us,cn,hk_sg
uv run equity pipeline --save-results
uv run equity dashboard build --output-dir site
```

Finnhub-enabled workflow:

```bash
export FINNHUB_API_KEY=your_key
uv run equity-news
uv run equity-sentiment
```

Loader examples with credentials:

```bash
uv run equity-loader list
uv run equity-loader show reddit_sentiment
uv run equity-loader show sec_filings
```

## Recommendation

Start with:

1. No keys at all for price ingestion and dashboard work.
2. Add `FRED_API_KEY` if you want macro data.
3. Add `FINNHUB_API_KEY` if you want news and sentiment.
4. Add Reddit and SEC settings only when you start using those loaders.
